# Acceptance Criteria

> Write acceptance criteria for every Must-Have user story before development starts.
> Use BDD format: **Given** [context], **When** [action], **Then** [outcome].

---

## US-01 — [User Story Title]

**Story:** As a [persona], I want to [action] so that [outcome].

**Happy Path**
- Given [initial context]
- When [user does action]
- Then [expected result]

**Edge Case 1**
- Given [edge condition]
- When [user does action]
- Then [expected result]

**Error Case**
- Given [error condition]
- When [user does action]
- Then [error is surfaced clearly, system remains stable]

## Definition of Done (applies to all stories)

A user story is done when:
- [ ] Code implemented and passing code review
- [ ] Unit tests written and passing
- [ ] Acceptance criteria above verified manually or via automated test
- [ ] CI pipeline is green
- [ ] No P0/P1 bugs open against this story
